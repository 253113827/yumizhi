# 贡献指南

感谢您有兴趣为 yumizhi.com 项目做贡献！本文档将指导您完成整个贡献过程。

## 项目结构

- `openclaw_skills.html` - 英文版技能集合页面
- `openclaw_skills_cn.html` - 中文版技能集合页面
- `improved_openclaw_skills.html` - 增强版技能集合页面
- `README.md` - 项目介绍
- `CHANGELOG.md` - 开发维护日志
- `CONTRIBUTING.md` - 本贡献指南
- `LICENSE` - 许可证信息

## 贡献方式

### 1. 技能贡献
如果您想添加新的OpenClaw技能：
- Fork 本仓库
- 在相应类别中添加新技能
- 提交 Pull Request
- 我们的团队将审核您的贡献

### 2. 功能改进
如果您想改进现有功能：
- 查看 Issues 页面寻找待解决问题
- 或提出新的功能建议
- Fork 仓库并进行修改
- 提交 Pull Request

### 3. 文档改进
如果您发现文档中有错误或可以改进的地方：
- Fork 仓库
- 修改相关文档
- 提交 Pull Request

## 开发流程

1. Fork 仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

## 代码风格

- 遵循一致的代码格式
- 添加适当的注释
- 确保代码可读性

## 联系方式

如有任何问题，请在 Issues 页面提出。
